## 完成事项

### Bug 修复

#### 1. 右键菜单目标路径错乱
- **原因**: `TreeNode.tsx` 完全没有 `onContextMenu` 处理，所有右键事件冒泡到容器 div，始终传入工作空间根路径 `rootPath`
- **修复**: 在 `TreeNode` 新增 `onContextMenu` prop 和 handler，调用 `e.preventDefault()` + `e.stopPropagation()` 阻止冒泡；`FileExplorer` 传入 `handleNodeContextMenu` 回调，正确传递当前节点的 `path`、`name`、`isDirectory`
- **文件**: [TreeNode.tsx](../src/renderer/components/explorer/TreeNode.tsx), [FileExplorer.tsx](../src/renderer/components/explorer/FileExplorer.tsx)

#### 2. 工作空间无法切换/关闭
- **原因**: 选择工作空间后缺少关闭/重置入口
- **修复**: 在资源管理器标题栏添加"关闭工作空间"按钮（`X` 图标），点击后调用 `setRootPath(null)` + `setTree([])` 回到空状态
- **文件**: [FileExplorer.tsx](../src/renderer/components/explorer/FileExplorer.tsx)

#### 3. Markdown 预览模式下 "-" 未正常显示
- **原因**: Tailwind CSS 4 preflight 重置了 `list-style: none`，`markdown.css` 未显式设置列表标记样式
- **修复**: 在 `markdown.css` 中为 `ul` 添加 `list-style: disc`，嵌套 `ul` 使用 `circle`/`square`，`ol` 使用 `list-style: decimal`
- **文件**: [markdown.css](../src/renderer/styles/markdown.css)

### 优化项

#### 4. 删除操作增加确认 + 放入回收站
- **当前**: 直接 `fs.rm` 永久删除，无确认
- **实现**: 
  - 新增 IPC 通道 `fs:trash`，在 IPC 层调用 `electron.shell.trashItem`（保持 service 层纯 Node.js 的架构原则）
  - 完整调用链：`ipcClient.trashItem()` → `preload` → `'fs:trash'` → `shell.trashItem`
  - 删除前弹出 `ConfirmDialog` 确认对话框
  - 降级策略：`trashItem` 失败（如跨卷移动）时自动降级到 `fs.rm`
- **文件**: [ipc-channels.ts](../src/shared/ipc-channels.ts), [file-system.ts](../src/main/ipc/file-system.ts), [preload/index.ts](../src/preload/index.ts), [ipc-client.ts](../src/renderer/lib/ipc-client.ts), [FileExplorer.tsx](../src/renderer/components/explorer/FileExplorer.tsx)

#### 5. 重命名输入框预填当前文件名
- **实现**: `handleRename` 中对非目录文件去掉扩展名（正则 `/\.[^.\\/]+$/`），输入框添加 `onFocus={(e) => e.target.select()}` 自动选中文本
- **文件**: [FileExplorer.tsx](../src/renderer/components/explorer/FileExplorer.tsx)

#### 6. 修复 expandedPaths 不可变性
- **原因**: `loadTree` 中 `expandedPaths.clear()` 直接修改 Zustand Store 的原始 Set，破坏不可变性
- **修复**: 改为 `useExplorerStore.setState({ expandedPaths: newExpanded })` 批量更新
- **文件**: [FileExplorer.tsx](../src/renderer/components/explorer/FileExplorer.tsx)

## 固化的架构模式

### 1. IPC 层使用 Electron API 的模式
`trashItem` 场景确立了"service 层不依赖 Electron，但 IPC 层可以直接使用 Electron API"的边界。当功能需要 Electron 专属 API（如 `shell.trashItem`）时，handler 直接在 IPC 层实现，不经过 service 层。

### 2. TreeNode 右键菜单模式
子组件（TreeNode）通过 `onContextMenu?: (e, node) => void` prop 接收右键回调，内部调用 `e.preventDefault()` + `e.stopPropagation()` 阻止冒泡。父组件（FileExplorer）负责将节点信息传给 `showMenu()`。

### 3. 删除确认模式
使用 `ConfirmDialog` 组件 + 条件渲染（`{deleteConfirmPath && <ConfirmDialog .../>}`）实现操作前确认。`handleDelete` 只设置状态触发对话框，真正的删除逻辑在 `confirmDelete` 中执行。

### 4. 文件名扩展名处理
使用正则 `/\.[^.\\/]+$/` 移除最后一个扩展名，兼容 Windows 路径中的反斜杠。

### 5. Zustand Set 批量更新
避免直接 mutating Store 中的 Set。使用 `setState({ expandedPaths: newExpanded })` 批量替换，保证不可变性。
