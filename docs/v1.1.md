- **原因**: `ContextMenu` 组件接收 `isDirectory` prop 但完全未使用，右键文件和文件夹显示相同菜单项（都有新建文件/文件夹/重命名/删除）

  - **文件夹菜单**: 新建文件、新建文件夹、粘贴（条件显示）、重命名、删除

  - `pathExists` 辅助函数检查目标是否存在，重名自动追加 ` - 副本` 后缀
  - 完整 IPC 链路：channel → service → handler → preload → ipc-client
  - 剪切板状态 `clipboardPath` 存储在 `useExplorerStore`
  - `FileExplorer` 新增 `handleCopy`（存入路径到剪切板）和 `handlePaste`（调用 `copyItem`、清除剪切板、刷新文件树）
- **文件**: [ipc-channels.ts](../src/shared/ipc-channels.ts), [file-ops.ts](../src/main/services/file-ops.ts), [file-system.ts](../src/main/ipc/file-system.ts), [preload/index.ts](../src/preload/index.ts), [ipc-client.ts](../src/renderer/lib/ipc-client.ts), [useExplorerStore.ts](../src/renderer/stores/useExplorerStore.ts), [FileExplorer.tsx](../src/renderer/components/explorer/FileExplorer.tsx)

## 固化的架构模式

### 1. 复制文件的命名冲突处理
`copyItem` 使用 `fs.access` 预检查目标是否存在，冲突时追加 ` - 副本` 后缀。使用 `fs.cp({ recursive: true })` 统一处理文件和目录复制，不区分类型。

### 2. 剪切板模式
应用内剪切板使用 Zustand Store 的 `clipboardPath` 字段，不涉及系统剪切板。复制只存路径，粘贴时通过 IPC 调用主进程的 `fs.cp` 完成实际文件复制。粘贴后自动清除剪切板并刷新文件树。

### 3. ContextMenu 条件渲染模式
使用 `isDirectory` 三元表达式控制两套完全不同的菜单项。`clipboardPath` 非空时文件夹菜单额外显示"粘贴"按钮，提供即时反馈。

### 4. 搜索按钮双位置模式
为确保搜索按钮始终可见：有打开文件时显示在 `EditorToolbar` 中，无文件时在 `EditorContainer` 空状态顶部渲染独立搜索按钮。两处共享同一个 `openSearch` action。
