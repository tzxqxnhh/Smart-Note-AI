# 迭代三 v1.2 报告

## 概述

迭代三实现 RAG（检索增强生成）模块，包括后端 4 个服务（Markdown 分块、SiliconFlow 嵌入、ChromaDB 向量存储、编排流水线）和前端 RAG 设置界面。在原有 95 个测试基础上新增 47 个测试，总计 142 个测试全部通过，无回归。

**日期**: 2026-06-15

---

## 完成事项

### 后端服务

#### 1. Chunker — Markdown 分块器
- **技术方案**: remark v15 AST 解析，以指定标题级别为边界切分，支持超大段落按段落边界/字符二次切分
- **核心参数**: `maxChunkSize`（最大字符数）、`maxOverlap`（重叠字符数）、`separator`（分隔符，如 `"##"`）
- **特性**: 前行内容作为 `(前言)` 前导块、`parentHeading` 追溯父标题、代码块内不分割、统一分配 chunkIndex/totalChunks
- **文件**: [chunker.ts](../src/main/services/chunker.ts)
- **测试**: 13 个（`tests/main/chunker.test.ts`）

#### 2. Embedder — SiliconFlow 嵌入 API 封装
- **技术方案**: 读取 `SILICONFLOW_API_KEY` 环境变量，调用 `https://api.siliconflow.cn/v1/embeddings`
- **模型**: Qwen/Qwen3-Embedding-8B
- **特性**: 32 条/批自动分片、重试 2 次（间隔 1s）、`Authorization: Bearer` 认证
- **初始化验证**: 发送测试嵌入请求验证 API Key 有效性
- **文件**: [embedder.ts](../src/main/services/embedder.ts)
- **测试**: 10 个（`tests/main/embedder.test.ts`）

#### 3. ChromaManager — ChromaDB 向量存储管理
- **连接**: `ChromaClient({ host: 'localhost', port: 8000 })`，集合名 `smart_note_demo`
- **操作**: upsertChunks、queryChunks（返回 id/content/metadata/distance）、deleteFileChunks（按 sourceFile 过滤）、resetCollection
- **关键修复**: `embeddingFunction` 传空操作对象而非 `null`，避免 chromadb 内部 `??` 穿透加载 DefaultEmbeddingFunction
- **文件**: [chroma-manager.ts](../src/main/services/chroma-manager.ts)
- **测试**: 10 个（`tests/main/chroma-manager.test.ts`）

#### 4. RagPipeline — RAG 编排
- **reindexFile**: 读取文件 → chunkMarkdown → embedder.embed → 删除旧块 → chromaManager.upsertChunks
- **reindexAll**: 递归扫描 .md 文件，逐个 reindexFile
- **ragQuery**: embedder.embedQuery → chromaManager.queryChunks（top-5）→ 返回 RagResponse
- **文件**: [rag-pipeline.ts](../src/main/services/rag-pipeline.ts)
- **测试**: 5 个（`tests/main/rag-pipeline.test.ts`）

### IPC 层

### 问题修复

#### 环境变量
- EMBEDDING_API_URL 未设置导致初始化报错 → 改为读取 `SILICONFLOW_API_KEY`
- 主进程入口新增 `loadEnvFile()` 加载项目根目录 `.env` 文件

#### ChromaDB DefaultEmbeddingFunction
- `embeddingFunction: null` 被 chromadb 内部 `??` 穿透 → 创建空操作 `EmbeddingFunction` 对象（`{ generate: async () => [] }`）
- ChromaClient 构造参数 `path` 已废弃 → 改用 `{ host, port }` 形式

#### GPU 报错
- GPU channel manager 虚拟化报错 → 添加 `--disable-gpu`
- DirectComposition `GetGpuDriverOverlayInfo` 报错 → 添加 `--disable-direct-composition`

## 固化的架构模式

### 1. 服务层单例模式
Embedder、ChromaManager、RagPipeline 均导出单例实例（`export const embedder = new Embedder()`），确保全应用共享同一连接和初始化状态。

### 2. RAG 流水线编排
```
读取 .md 文件 → chunkMarkdown(remark AST) → embedder.embed(SiliconFlow)
→ chromaManager.deleteFileChunks(旧数据) → chromaManager.upsertChunks(新数据)
```

### 3. 覆盖层 UI 模式
RagSettingsPanel 和 SearchBar 共享相同的覆盖层模式：
- `fixed inset-0 z-40 bg-black/40` 半透明遮罩
- 内部居中卡片 `w-[480px]`，暗色主题
- Header（图标 + 标题 + 关闭按钮）+ Body（表单）+ Footer（取消/确定按钮）

### 4. 环境变量加载顺序
主进程入口在模块加载后立即调用 `loadEnvFile()` 解析 `.env` → 注入 `process.env` → 后续服务模块在 `initialize()` 时读取。
