# 迭代四 v1.3 报告

## 概述

迭代四实现向量库管理功能，新增浏览、查看、删除 ChromaDB 中已索引文档切片的能力。入口位于 Agent 面板，通过 Database 按钮切换视图。在原有 142 个测试基础上新增 50 个测试，总计 192 个测试全部通过，无回归。

**日期**: 2026-06-17

## 完成事项

### 后端服务

#### 1. ChromaManager 扩展 — 向量库管理方法
- **listFileChunks()**: 调用 `collection.get()` 获取全部切片，按 `sourceFile` 分组返回 `FileChunkGroup[]`，contentPreview 截取前 10 字符
- **getChunkDetail(id)**: 调用 `collection.get({ ids: [id] })` 返回完整切片详情
- **deleteChunks(ids)**: 调用 `collection.delete({ ids })` 批量删除，空数组直接返回
- **getChunkCount()**: 调用 `collection.count()` 返回切片总数
- **storedAt 时间戳**: `upsertChunks()` metadata 中新增 `storedAt: new Date().toISOString()`
- **文件**: [chroma-manager.ts](../src/main/services/chroma-manager.ts)
- **测试**: 新增 10 个（`tests/main/chroma-manager.test.ts`，从 10 增至 20）

## 固化的架构模式

### 1. 组件内嵌模式
VectorDbPanel 内嵌在 AgentPanel 中，通过 store 的 `isOpen` 字段条件切换，不新增独立面板区域，复用右侧栏空间。

### 2. ChunkDetailModal 注册模式
详情弹窗在 App.tsx 作为同级组件注册（`z-50`），脱离 AgentPanel 的 DOM 层级以确保正确的 z-index 覆盖。

### 3. 批量操作模式
使用 `batchMode` 布尔值 + `selectedChunkIds` Set 管理批量删除状态，进入/退出批量模式时清空勾选，删除后自动刷新列表。

### 4. 浅合并陷阱
Zustand 5 `setState` 做浅合并而非替换，组件测试中需要显式设置所有状态字段（特别是 `error: null`），避免前测试状态泄露。
