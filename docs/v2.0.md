## 一、概述

v2.0 修复了 v1.5 中 4 个核心 Bug，重构了 Agent 预设操作的触发机制。将文本操作（总结/扩写/格式化）从 PresetButtons 迁移到编辑器右键菜单，将结构图生成入口迁移到文件资源管理器右键菜单，精简了 Agent 底部操作区。同时新增向量库开关和模型切换能力。

本迭代遵循 TDD，在原有 251 个测试基础上净增 11 个，总计 262 个测试，24 个测试文件全部通过，无回归。

## 二、修复清单

### 1. Bug 1 — 编辑器选区与 Agent 联动断裂 ✓

| 项目 | 说明 |
|------|------|
| 根因 | `CodeEditor` 组件未暴露文本选中事件，`setSelectedText()` 无任何调用方 |
| 修复 | CodeEditor 新增 `EditorView.updateListener` 监听选区变化；EditorContainer 新增编辑器右键菜单（总结/扩写/格式化） |
| 表现 | 选中文本 → 右键 → 总结/扩写/格式化 → Agent 输出结果 |

### 2. Bug 2 — 结构图入口缺失 ✓

| 项目 | 说明 |
|------|------|
| 根因 | `selectedFolderPath` 从未被设置，文件资源管理器右键菜单无「生成结构图」选项 |
| 修复 | ContextMenu 新增 `onVisualize` prop 和「生成结构图」菜单项；FileExplorer 新增 `handleVisualize` handler |
| 表现 | 右键文件夹 → 生成结构图 → Agent 输出 Mermaid 代码 |

### 3. Bug 3 — toolStep 重复渲染 ✓

| 项目 | 说明 |
|------|------|
| 根因 | `llm:stream-end` 中重复发送了已在 `llm:stream-chunk` 推送过的 searchStep |
| 修复 | 移除 `llm:stream-end` 中的 `toolSteps` 字段；`useAgentStore.onLlmStreamEnd` 不再从 `data.toolSteps` 合并 |

### 4. Bug 4 — 向量检索内容传递诊断 ✓

| 项目 | 说明 |
|------|------|
| 根因待确认 | ChromaDB documents 为空 / token 截断 / prompt 格式 |
| 措施 | 在 `llm.ts` 添加 chunk content 诊断日志；将 `buildRagPrompt` 提取到 `rag-pipeline.ts` 方便调优 |

## 三、架构重构

### PresetButtons 精简

| 移除 | 新增 |
|------|------|
| 总结、扩写、格式化、结构图、问答 | 向量库开关（Database 图标，绿色/灰色区分状态） |
| | 清空对话（Trash2 图标） |

### AgentPanel 顶部栏精简

- 移除「清空对话」按钮（迁移至 PresetButtons）
- 保留「向量库管理」入口按钮（简化图标为 `DB` 文字）

### 向量库开关链路

```
渲染进程                             主进程
─────────                           ─────────
useAgentStore.vectorDbEnabled
    ↓
ipcClient.llmChatStream(query, id, vectorDbEnabled)
    ↓
preload (IPC send) ──────────────→  llm.ts handler
                                      ├─ true:  RAG 检索 → 构建上下文 → LLM 流式生成
                                      └─ false: 跳过 RAG → 通用 prompt → LLM 流式生成
```

### buildRagPrompt 提取

- 从 [llm.ts](src/main/ipc/llm.ts) 内联 prompt → [rag-pipeline.ts](src/main/services/rag-pipeline.ts) 独立导出函数
- `ragQueryWithLLM` 同步使用 `buildRagPrompt`
- 方便单独测试和 prompt 调优
