# 迭代1.5 v1.5 报告

## 概述

迭代1.5 实现了 DeepSeek LLM 集成与流式对话，让 Agent 真正能对笔记内容进行流式问答。新增工具调用步骤展示（Read/Search/Write），复刻 Claude Code 的操作过程可视化体验。在原有 196 个测试基础上新增 55 个，总计 251 个测试全部通过，无回归。

**日期**: 2026-06-17

---

## 完成事项

### 1. DeepSeek LLM 客户端 — llm-client.ts

- 重写 `llm-client.ts`，将 `OllamaClient`/`AnthropicClient` 骨架替换为 `DeepSeekClient`
- API 地址：`https://api.deepseek.com/v1/chat/completions`（OpenAI 兼容接口）
- 实现非流式 `chat(messages, options?)` 和流式 `chatStream(messages, options?)` 两个方法
- 流式使用 `ReadableStream` reader + SSE 解析（`data: [DONE]` 终止）
- 默认模型 `deepseek-chat`（环境变量 `DEEPSEEK_MODEL` 可覆盖）、temperature 0.3、max_tokens 4096
- `LLMProvider` 接口从 `generate(prompt)` 改为 `chat(messages[])`（Chat Completions 风格）
- 导出 `deepseekClient` 单例
- **文件**: [llm-client.ts](../src/main/services/llm-client.ts)

### 2. RAG + LLM 问答管道 — rag-pipeline.ts

- 新增 `ragQueryWithLLM(query)` 方法：嵌入查询 → ChromaDB Top-K 检索 → 构建 system prompt（含上下文）→ DeepSeek 生成 → 返回带引用的答案
- 原有 `ragQuery()` 保留不变（向后兼容）
- System prompt 使用上下文注入 + 引用格式要求
- **文件**: [rag-pipeline.ts](../src/main/services/rag-pipeline.ts)

### 3. LLM IPC Handler 重写 — llm.ts

- `setupLLMHandlers` 改为接受 `mainWindow: BrowserWindow` 参数
- `llm:chat` 改为 `ipcMain.on`（非 `handle`）：RAG 检索 → toolStep 推送 → 流式 LLM 生成 → 逐 token 推送 → 流结束
- `llm:summarize`/`llm:expand`/`llm:format`：接收选中文本 → 构造 prompt → DeepSeek 生成 → 返回结果
- `llm:generate-tree`：接收文件夹路径 → 扫描目录结构 → DeepSeek 生成 Mermaid 代码
- 新增 `buildTreeString()` 辅助函数
- **文件**: [llm.ts](../src/main/ipc/llm.ts)

### 4. IPC 注册和入口更新

- `setupIPC()` 接受 `mainWindow` 参数传递给 `setupLLMHandlers`
- `createWindow()` 返回 `BrowserWindow` 引用
- `app.whenReady()` 中先创建窗口，再注册 IPC（顺序调整）
- **文件**: [index.ts](../src/main/ipc/index.ts), [index.ts](../src/main/index.ts)

### 5. 流式事件通道

- 新增 2 个 IPC 通道：`llm:stream-chunk`（主→渲染逐 token）、`llm:stream-end`（流结束含引用）
- Preload 新增 `onLlmStreamChunk`/`onLlmStreamEnd` 监听器、`llmChatStream` 发送方法
- ipc-client 新增 `LlmStreamChunkData`/`LlmStreamEndData` 类型和封装函数
- **文件**: [ipc-channels.ts](../src/shared/ipc-channels.ts), [preload/index.ts](../src/preload/index.ts), [ipc-client.ts](../src/renderer/lib/ipc-client.ts)

### 6. useAgentStore 重写

- 新增 `streamingMessage`、`toolSteps` 状态管理
- `sendQuery`：添加用户消息 → 注册流式监听 → 发起流式请求 → chunk 追加 → end 固化
- `runPresetAction`：summarize/expand/format 调用对应 IPC，visualize 调用 generateTree
- 错误处理：API Key 未设置、网络错误、操作失败
- 新增 `resetAgentStore()` 导出（测试用）
- **文件**: [useAgentStore.ts](../src/renderer/stores/useAgentStore.ts)

### 7. ChatMessage 组件更新

- 新增 `isStreaming` prop，流式时显示闪烁光标（`animate-pulse`）
- 新增 `ToolStepItem` 子组件，支持 read/search/write 三种类型
- toolStep 列表在消息正文上方渲染，可点击展开/折叠详情
- search 类型展示切片来源、内容预览
- **文件**: [ChatMessage.tsx](../src/renderer/components/agent/ChatMessage.tsx)

### 8. AgentPanel 流式集成

- 读取 `streamingMessage` 并渲染为带闪烁光标的 ChatMessage
- 流式消息与固化消息共存于消息列表中
- **文件**: [AgentPanel.tsx](../src/renderer/components/agent/AgentPanel.tsx)

### 9. 新增类型

- `ToolStep` 接口：id, type (read/search/write), status, filePath, chunkCount, chunkSources, writtenPath
- `ChatMessage` 新增 `toolSteps?: ToolStep[]` 字段
- **文件**: [types.ts](../src/shared/types.ts)
